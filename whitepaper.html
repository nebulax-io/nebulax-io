Section 01

Executive Summary

NEBULAX is a decentralized physical infrastructure network (DePIN) designed to aggregate idle computing resources from devices worldwide — smartphones, laptops, desktops, and servers — into a unified, on-demand AI compute marketplace.

The global AI industry faces an acute compute shortage. Training a single large language model can cost $50–100 million, and inference demands are growing exponentially. Meanwhile, billions of devices sit idle, their processing power wasted.

NEBULAX bridges this gap. Device owners share their idle GPU/CPU resources and earn $NBX tokens. AI companies and researchers access affordable, scalable compute without massive upfront infrastructure investment.

Section 02

The Problem

2.1 The AI Compute Crisis

The demand for AI compute is doubling every 6 months, while supply grows at only 20% annually. This creates a widening gap that threatens to slow AI innovation and concentrate power in the hands of a few tech giants.

2.2 Centralization Risk

Currently, 80% of AI compute is controlled by just 5 companies (AWS, Google Cloud, Azure, Oracle, IBM). This creates:

  • Single points of failure — outages affect millions
  • Price manipulation — no competition means inflated costs
  • Censorship risk — platforms can deny service arbitrarily
  • Geographic concentration — mostly US/EU, underserving Global South

2.3 Wasted Resources

Globally, an estimated 85% of consumer GPU capacity sits idle at any given time. A gaming PC used 4 hours a day wastes 20 hours of potential compute. Multiply this by billions of devices, and the untapped capacity exceeds what all cloud providers offer combined.

Key Insight

The world doesn't lack computing power — it lacks a network to organize it. NEBULAX is that network.

Section 03

Our Solution

3.1 The NEBULAX Network

NEBULAX creates a two-sided marketplace:

  • Supply Side: Device owners who share idle compute resources
  • Demand Side: AI companies, researchers, and developers who need compute

3.2 Core Components

One-Click Node Software

Lightweight application for Windows, macOS, Linux, Android, and iOS. Installation takes under 60 seconds. The software automatically detects hardware capabilities, optimizes resource allocation, and connects to the NEBULAX network.

AI Task Marketplace

A decentralized marketplace where compute buyers post tasks (model training, inference, rendering) and the network automatically matches them with suitable nodes based on hardware requirements, geographic proximity, and reliability scores.

AI Agent Hub

A platform for creating, trading, and deploying autonomous AI agents. Users can build agents that work 24/7, generating passive income from the emerging agent economy.

Stake-to-Compute

Users can stake $NBX tokens to access compute credits without owning powerful hardware. This democratizes AI access for developers and researchers in resource-constrained environments.

Section 04

How It Works

4.1 Node Operation

  1. Install: User installs the NEBULAX node application
  2. Register: Device capabilities are benchmarked and registered on-chain
  3. Connect: Node joins the network and begins receiving tasks
  4. Earn: $NBX tokens are distributed based on work completed

4.2 Task Distribution

When a compute buyer submits a task, the network's matching algorithm considers:

  • Hardware requirements — GPU model, VRAM, CPU cores
  • Reliability score — uptime history, task completion rate
  • Geographic location — latency optimization
  • Stake amount — higher stake = priority matching

4.3 Verification & Payment

Completed tasks are verified through a multi-step process:

  1. Proof of Compute: Node submits cryptographic proof of work done
  2. Cross-validation: Random nodes verify the output
  3. Payment: $NBX tokens are released to the provider
Reward = Base_Rate × Task_Complexity × Reliability_Score × Stake_Multiplier
Section 05

Network Architecture

5.1 Multi-Chain Design

NEBULAX operates across multiple blockchains for scalability and resilience:

Layer Chain Purpose
Settlement Ethereum L1 Token, governance, staking
Execution Base (L2) Task matching, payments
High-speed Solana Microtransactions, real-time data
Storage Arweave Task logs, proofs

5.2 Consensus Mechanism

NEBULAX uses a novel Proof-of-Compute (PoC) consensus:

  • Nodes must prove they actually performed the requested computation
  • Validators cross-check results using redundant computation
  • Disputes are resolved through a decentralized arbitration system
  • Staked tokens act as collateral against malicious behavior

5.3 Security Model

The network employs multiple security layers:

  • Data encryption — All task data is encrypted end-to-end
  • Sandboxed execution — Tasks run in isolated environments
  • Reputation system — Malicious nodes are penalized and removed
  • Bug bounty — $500K+ bounty program for vulnerability discovery
